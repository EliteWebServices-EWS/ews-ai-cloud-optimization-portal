Security & Trust

How Elite Web Services protects your AWS environment during assessments and advisory engagements.

Read-Only Cross-Account Access

Our portal uses AWS STS AssumeRole with a cross-account IAM role. We request read-only permissions only — no write access, no access keys stored, and no persistent credentials on our servers.

External ID Protection

Every customer connection uses a unique External ID to prevent the confused deputy problem. Your IAM trust policy must include this ID before we can assume your role.

What We Access

Cost & Usage

AWS Cost Explorer and billing data to identify optimization opportunities.

Resource Inventory

EC2, RDS, S3, and other service metadata — configuration only, never your data payloads.

Security Posture

Security Hub and Config findings to surface misconfigurations and compliance gaps.

Well-Architected

Framework-aligned reviews across operational excellence, security, reliability, performance, and cost.

What We Never Access

  • Application data stored in databases, S3 objects, or EBS volumes
  • Secrets Manager or Parameter Store values
  • IAM user passwords or long-lived access keys
  • Ability to create, modify, or delete resources in your account

AI Insights

Claude AI is used for generating cost and architecture insights only. Aggregated, non-sensitive metadata is sent for analysis — never raw credentials, secrets, or customer data payloads.

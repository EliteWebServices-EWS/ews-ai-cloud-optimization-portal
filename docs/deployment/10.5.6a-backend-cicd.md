# Sprint 10.5.6A — Backend CI/CD Automation

## Status

In Progress

## Purpose

Create a dedicated GitHub Actions workflow that validates, tests, builds, deploys, and smoke-tests the SISU'M backend independently of the frontend deployment.

## Workflow File

`.github/workflows/deploy-backend.yml`

## Trigger Strategy

The workflow runs:

- Automatically after backend changes are merged into `main`
- Manually through `workflow_dispatch`

Automatic deployment is limited to changes under:

- `backend/**`
- `.github/workflows/deploy-backend.yml`

Frontend-only changes do not trigger the backend deployment.

## Authentication

GitHub Actions authenticates with AWS through OpenID Connect.

Deployment role:

`SisumBackendDeployRole`

AWS Region:

`us-east-1`

No long-lived AWS access keys are stored in GitHub.

## Pipeline Stages

1. Check out the repository
2. Install Node.js 24
3. Install backend dependencies
4. Compile TypeScript
5. Run backend tests
6. Install AWS SAM CLI
7. Validate the SAM template
8. Build the SAM application
9. Assume the AWS deployment role through OIDC
10. Verify the assumed AWS identity
11. Deploy the backend with AWS SAM
12. Verify the CloudFormation stack
13. Smoke-test the live health endpoint

## Deployment Target

- Stack: `sisum-backend`
- Region: `us-east-1`
- Environment: `production`
- Provider mode: `mock`
- Lambda function: `sisum-backend-production`

## Security Controls

- `contents: read`
- `id-token: write`
- Production GitHub environment
- Dedicated deployment role
- Expected AWS account restriction
- No static AWS credentials
- Backend and frontend deployment workflows remain separate

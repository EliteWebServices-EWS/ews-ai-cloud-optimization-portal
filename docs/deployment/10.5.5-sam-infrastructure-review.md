## Risks and Limitations

1. **Local SAM on Windows x86 + arm64** â€” cold starts can exceed 30s; use direct Node handler testing or test in CI/Linux/AWS after deploy.
2. **Unbundled artifact** â€” `NodejsNpmBuilder` packages full `node_modules` (including unused AWS SDK clients); increases cold-start time. Consider esbuild bundling in a future sprint once esbuild is reliably on PATH in CI.
3. **`sam build --clean`** â€” not available in SAM CLI 1.162.1; manual `.aws-sam` cleanup required on Windows when rebuild fails.
4. **External IAM role** â€” `SisumLambdaExecutionRole` must exist in the target account with CloudWatch Logs permissions before deploy.
5. **Express CORS** â€” echoes a single `CORS_ORIGIN`; API Gateway handles multi-origin preflight.
6. **Legacy `backend/src/`** â€” old Portal handler remains in repo but is excluded from TypeScript build and no longer referenced by SAM.

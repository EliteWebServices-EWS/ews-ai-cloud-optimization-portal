# Sprint 10.5.12 — Role-Based Access Control (RBAC) Authorization

## Objective

Implement production-grade Role-Based Access Control (RBAC) for the SISU'M Cloud Optimization Platform using Amazon Cognito groups and API Gateway JWT authorization.

This sprint ensures that authenticated users receive permissions based on their assigned Cognito group instead of allowing unrestricted access to protected backend APIs.

---

# Sprint Goals

- Protect backend APIs using Amazon Cognito JWTs
- Read user identity from JWT claims
- Extract Cognito groups from authenticated users
- Enforce authorization using Express middleware
- Restrict workflow execution to authorized roles
- Validate RBAC using real Cognito users

---

# Authorization Model

Three Cognito groups are supported.

| Role | Description |
|------|-------------|
| viewer | Read-only dashboard access |
| analyst | Can execute optimization workflows |
| admin | Full platform administration |

## Authenticated

All authenticated users may access read-only APIs.

Examples include:

```
GET /providers
GET /plugins
GET /reports
GET /recommendations
GET /confidence
GET /financial/summary
```

Required role:

```
viewer
analyst
admin
```

---

## Analyst/Admin Only

The following operations require elevated permissions.

```
POST /workflows/run
POST /execution/simulate
POST /reports/generate
```

Allowed roles:

```
analyst
admin
```

Viewer access is denied.

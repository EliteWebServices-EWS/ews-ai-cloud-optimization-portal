**Project:** EWS AI Cloud Optimization Portal / SISU'M

**Environment:** Production

**AWS Region:** us-east-1

**Status:** In Progress

# Objective

The objective of Sprint 10.5.11 is to secure the SISU'M Cloud Optimization Portal by introducing enterprise-grade authentication and authorization using Amazon Cognito and API Gateway JWT authorization.

This sprint transitions the application from a publicly accessible demonstration platform to a secure production-ready platform where only authenticated users can access protected functionality.

Authentication will be implemented first, followed by authorization, frontend route protection, API hardening, and Role-Based Access Control (RBAC).

# Authentication Architecture

The authentication workflow follows the OAuth 2.0 Authorization Code Grant with PKCE.

Flow:

```text
User opens dashboard
        │
        ▼
Frontend checks session
        │
        ▼
No session
        │
        ▼
Redirect to Cognito
        │
        ▼
User signs in
        │
        ▼
Password change (first login only)
        │
        ▼
Authorization Code
        │
        ▼
Frontend exchanges code
        │
        ▼
JWT Tokens
        │
        ▼
Dashboard loads
        │
        ▼
JWT attached to API requests
```

# Lessons Learned

Managed Login Version 2 requires Managed Login Branding.

CloudFormation must own the branding resource to prevent deployment conflicts.

Administrator accounts should always complete first login before backend authorization is enabled.

Authentication should be validated before protecting production APIs.
